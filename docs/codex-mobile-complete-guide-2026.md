# 【保存版】Codexをスマホで使うための設定方法と実践ガイド

最終更新日: 2026年5月15日  
対象: ChatGPTアプリ版Codex Mobile、Codexデスクトップアプリ、スマホからの開発・レビュー・業務操作

## この記事の結論

Codex Mobileは、スマホ単体の開発環境ではありません。

スマホのChatGPTアプリから、MacやWindows、devbox、リモート環境で動いているCodexへ接続し、外出先から作業を確認・指示・承認・修正できる機能です。

つまり、スマホ側は「司令室」、PC側は「実行環境」です。

スマホからできることは、単なるリモート操作ではありません。

- 進行中のCodexスレッドを確認する
- Codexからの質問に答える
- コマンド実行やファイル変更を承認する
- 作業方針を変える
- 新しい作業を開始する
- 差分、テスト結果、スクリーンショット、ターミナル出力を確認する
- 複数スレッドの進行をスマホから管理する

OpenAI公式発表でも、Codex in the ChatGPT mobile appはiOS/Android向けにプレビュー展開され、Codexが動いているマシンのライブ状態をスマホへ読み込む体験として説明されています。

この記事では、初期設定、詰まりやすいポイント、スマホ運用時の注意点、実践的な使い方、プロンプト集、Claude Coreワークのディスパッチとの比較までまとめます。

## 大前提: Codex Mobileはスマホ単体で動くものではない

まず、ここを理解しておくと迷いません。

Codex Mobileは、スマホの中だけでCodexを動かす機能ではありません。

仕組みは次のイメージです。

```text
スマホのChatGPTアプリ
  ↓
Codex Mobile
  ↓
Mac / Windows / devbox / リモート環境で動いているCodex
  ↓
その環境内のファイル、ターミナル、ブラウザ、アプリ、プロジェクト
```

そのため、スマホ側にCodexの画面が出ていても、接続先のCodexデスクトップアプリや実行環境が準備できていないと進みません。

よく出る画面がこれです。

```text
デスクトップアプリを待っています
```

この表示が出たら、スマホで待ち続けるのではなく、MacやWindows側のCodexアプリへ移動してください。

## まず確認すること

設定前に、次の4つを確認してください。

| 確認項目 | 理由 |
|---|---|
| PCにCodexデスクトップアプリが入っているか | スマホはPC側のCodexへ接続するため |
| Codexデスクトップアプリが最新版か | Codex Mobile連携が古いバージョンに出ない可能性があるため |
| スマホのChatGPTアプリが最新版か | Codex Mobileの項目や接続画面が出ない可能性があるため |
| PCとスマホが同じChatGPTアカウントか | アカウントが違うと接続先が見えないため |

個人アカウント、Workspaceアカウント、Business/Enterprise/Eduアカウントを使い分けている人は、ここでズレやすいです。

## パターンA: Codexデスクトップアプリをまだ入れていない人

まず、PC側にCodexデスクトップアプリを入れます。

1. PCでCodexデスクトップアプリをダウンロードする
2. インストールする
3. Codexアプリを開く
4. ChatGPTアカウントでログインする
5. 作業したいフォルダを選ぶ
6. 必要ならプロジェクトを作成する

ここまでできたら、スマホ側の設定へ進みます。

## パターンB: すでにCodexデスクトップアプリを入れている人

すでにCodexアプリを入れている人は、まず最新版かどうかを確認してください。

MacやWindowsのCodexアプリ右上に、

```text
更新
```

のような表示が出ている場合があります。

これが出ている場合は、先に更新してください。

ここを押していないだけで、スマホ側では「デスクトップアプリを待っています」のまま進まないことがあります。

見落としやすいですが、かなり重要です。

## Codex Mobileの接続手順

ここからは、Codexアプリをすでに入れている人も、今インストールした人も共通です。

### 手順1: スマホのChatGPTアプリを最新版にする

App StoreまたはGoogle PlayでChatGPTアプリを開き、アップデートが出ていれば更新します。

Codex MobileはChatGPTアプリ内から使うため、アプリが古いとCodexの項目が表示されなかったり、接続画面から進まなかったりする可能性があります。

### 手順2: スマホ側で「Codex」を開く

ChatGPTアプリを開き、メニュー内の「Codex」を押します。

Codex Mobileを接続するための案内が出たら、

```text
Codex Mobileを接続する
```

のようなボタンを押します。

### 手順3: 「デスクトップアプリを待っています」と出たらPC側へ行く

ここで、多くの人が次の表示になります。

```text
デスクトップアプリを待っています
```

この表示が出たら、スマホ側で待つのではなく、PC側のCodexアプリへ移動します。

### 手順4: PC側の左メニューにある「Codex mobileを設定」を押す

Codexデスクトップアプリの左側メニューに、

```text
Codex mobileを設定
```

のような項目が表示されているはずです。

これを押します。

もし表示されない場合は、次を確認してください。

- Codexデスクトップアプリを最新版にしているか
- 右上の「更新」を押したか
- スマホと同じChatGPTアカウントでログインしているか
- Codexデスクトップアプリを再起動したか
- スマホのChatGPTアプリを最新版にしたか

### 手順5: PC側で「始める」を押す

「Codex mobileを設定」を押すと、PC側にCodex Mobileのセットアップ画面が出ます。

中央付近に、

```text
始める
```

というボタンがあるので押します。

この画面では、スマホからCodexの作業を続けられること、完了通知や確認通知を受け取れること、新しい作業を開始できることなどが説明されます。

### 手順6: PC側で「許可」を押す

次に、

```text
お使いのスマートフォンでこのコンピュータを操作できるようにする
```

のような許可画面が出ます。

ここで「許可」を押します。

この許可をしないと、スマホからPC側のCodexへアクセスできません。

### 手順7: QRコードをスマホで読み取る

許可を押すと、PC側にQRコードが表示されます。

スマホ側のChatGPTアプリからQRコードを読み取ります。

iOSとAndroidの選択が出る場合は、自分のスマホに合わせて選びます。

### 手順8: スマホ側で「続ける」または「Authorize this phone」を押す

QRコードを読み取ると、スマホ側にサインイン確認や端末承認の画面が出ます。

```text
続ける
```

または、

```text
Authorize this phone
```

を押します。

### 手順9: 「接続されました」と出たら成功

スマホ側で承認すると、PC側に、

```text
接続されました
```

と表示されます。

これでCodex Mobileの接続は成功です。

## 接続後に出てくる3つの設定

接続後、PC側に追加設定が表示されることがあります。

主に次の3つです。

1. このコンピューターをスリープさせない
2. パソコンでの利用を有効にする
3. Chrome拡張機能をインストール

自分が所有している信頼できるPCで使う場合は、基本的にはすべて有効化がおすすめです。

### 1. このコンピューターをスリープさせない

Codex Mobileは、PC側のCodexをスマホから操作する仕組みです。

PCがスリープすると、スマホ側から操作できなくなる可能性があります。

外出先から長めの作業を任せたい人は、有効にしておくのがおすすめです。

ただし、バッテリー駆動のノートPCでは電池消費が増えます。

外出前に電源接続、放熱、ネットワーク安定性も確認してください。

### 2. パソコンでの利用を有効にする

これは、CodexがPC上のアプリや画面を操作できるようにする設定です。

アプリ起動、画面確認、ブラウザ操作、ローカル環境での確認などをCodexに任せたい場合に関係します。

開発だけでなく、Web操作や制作補助にも使うなら有効化しておくと便利です。

### 3. Chrome拡張機能をインストール

Webサイトの確認、フォーム入力、ブラウザ上の検証、管理画面の操作などをCodexに任せたい場合に使います。

Webアプリ開発、LP確認、管理画面操作、調査系タスクをスマホから投げたい人は入れておく価値があります。

## 全部許可してよい条件

次の条件に当てはまるなら、基本的には全部有効化でOKです。

- 自分が所有しているPCである
- ChatGPT/Codexにログインしているアカウントを自分で管理している
- PC内にCodexへ触らせてよい作業フォルダを分けている
- 削除、公開、送信、課金、force pushなどは必ず人間承認にしている
- 会社や学校の管理端末ではない

逆に、次の環境では慎重にしてください。

- 共有PC
- 会社の管理端末
- 顧客データや個人情報が大量に入っている端末
- 家族や他人も使う端末
- セキュリティポリシーが厳しい組織の端末

CodexはPC内のファイル、アプリ、ブラウザ、ターミナルにアクセスする可能性があります。

便利さと権限はセットです。

## スマホでCodexを動かすときに気をつけるべきこと

スマホ運用では、PC前で使うときとは違う注意点があります。

### 1. 画面が小さいので、差分確認を雑にしない

スマホは差分やログを読むには画面が小さいです。

小さな修正ならスマホ承認でもよいですが、次の作業はPCで確認するか、Codexに要約とリスク説明を出させてから承認してください。

- 認証まわりの変更
- 決済まわりの変更
- データ削除処理
- DBマイグレーション
- 本番環境への反映
- 外部送信
- SNSやnoteへの公開

スマホでは「見たつもり」が起きやすいです。

### 2. 指示は短くても、完了条件は明確にする

スマホ入力は短文になりがちです。

しかし、Codexに任せるときは、最低限この3つを入れると安定します。

- 目的
- 触ってよい範囲
- 完了条件

例:

```text
READMEの導入手順を初心者向けに直して。
コードは触らないで。
完了後に変更箇所と確認した観点を3行でまとめて。
```

### 3. 長時間タスクは「途中報告」を指定する

スマホから投げる作業は、移動中や外出中に進むことが多いです。

そのため、最初から途中報告を指定しておくと安心です。

```text
まず現状調査して、方針を短く報告してから修正して。
30分以上かかりそうなら、途中で進捗を教えて。
```

### 4. 外出前にPC側の状態を整える

Codex MobileはPC側が実行環境です。

外出前に次を確認しておくと、外からの成功率が上がります。

- PCが電源につながっている
- Wi-Fiが安定している
- Codexデスクトップアプリが起動している
- 作業リポジトリが開ける状態
- 必要な環境変数や認証が入っている
- Dockerやdev serverが必要なら起動できる状態
- スリープしない設定が有効

### 5. 秘密情報をスマホで雑に打たない

APIキー、パスワード、顧客情報、個人情報をスマホ入力欄へ直接貼るのは避けてください。

必要な場合は、PC側の環境変数、秘密情報管理、既存の設定ファイル、Codexの安全なAPIキー作成フローなどを使います。

### 6. 承認が必要な操作を最初に宣言する

スマホ運用では、うっかり承認が起きやすくなります。

最初にこう言っておくと安全です。

```text
削除、上書き、外部送信、公開、課金、本番反映、force pushは、実行前に必ず確認して。
```

### 7. 通信が切れる前提で「再開しやすい依頼」にする

移動中は通信が不安定です。

Codexには、途中経過が残る形で作業させると再開しやすくなります。

```text
調査メモをdocs/worklog.mdに残しながら進めて。
詰まったら、次に試すことを箇条書きで残して止まって。
```

### 8. スマホからは「指示」と「承認」に集中する

スマホで細かいコードレビューを全部やるより、Codexに次を出させると実用的です。

- 何を変えたか
- なぜ変えたか
- リスクは何か
- どう検証したか
- PCで見るべきファイルはどこか

スマホは細部を全部読む場所というより、作業を前に進めるための判断場所として使うと強いです。

## よくある詰まりポイント

### 1. スマホ側で待っているだけ

「デスクトップアプリを待っています」と出たら、PC側へ移動します。

PC側の左メニューにある「Codex mobileを設定」を押してください。

### 2. PCにCodexアプリを入れていない

Codex Mobileは、PC側のCodexへ接続する機能です。

PCにCodexデスクトップアプリがない場合は、先にインストールが必要です。

### 3. Codexデスクトップアプリを更新していない

右上に「更新」が出ていたら、必ず押してください。

更新後にCodexアプリを再起動すると改善することがあります。

### 4. スマホとPCで違うアカウントにログインしている

個人アカウント、Workspace、Business、Enterprise、Eduを使い分けている人は要注意です。

スマホのChatGPTアプリとPCのCodexアプリが同じアカウントか確認してください。

### 5. PCがスリープしている

PCがスリープすると、スマホ側から操作できない可能性があります。

接続後は「このコンピューターをスリープさせない」を有効にするのがおすすめです。

### 6. VPNや社内ネットワークで接続が不安定

VPN、社内プロキシ、MDM管理、ファイアウォールにより接続が不安定になる場合があります。

会社端末では、情シスや管理者のルールを確認してください。

## 最短手順まとめ

1. スマホのChatGPTアプリを最新版にする
2. スマホ側で「Codex」を押す
3. 「Codex Mobileを接続する」を押す
4. 「デスクトップアプリを待っています」と出たらPC側へ行く
5. PCにCodexアプリがない人はダウンロードする
6. PCのCodexアプリを開く
7. 右上に「更新」が出ていたら押す
8. PC左側の「Codex mobileを設定」を押す
9. 「始める」を押す
10. 「許可」を押す
11. 表示されたQRコードをスマホで読み取る
12. スマホ側で「続ける」または「Authorize this phone」を押す
13. PC側に「接続されました」と出たら成功
14. 接続後の追加設定は、信頼できる自分のPCなら基本的に有効化する

## スマホCodexはどう使えばいいか

スマホCodexは、PCの代わりに全部をやる道具というより、作業を止めないための道具です。

特に相性がよいのは、次のような使い方です。

### 1. 外出中に調査を進める

```text
このリポジトリの認証まわりを調査して、ログイン処理の流れ、関連ファイル、リスクをまとめて。
まだコード変更はしないで。
```

移動中に調査を走らせ、到着後にPCで実装する流れが作れます。

### 2. 小さな修正を任せる

```text
READMEの誤字と古い手順を直して。
コードは触らず、Markdownだけ変更して。
最後に変更点を3つにまとめて。
```

ドキュメント修正、軽い文言調整、テスト追加などはスマホからでも回しやすいです。

### 3. 実装方針だけ作らせる

```text
この機能を追加する場合の実装方針を立てて。
既存設計を読んで、触るファイル、リスク、テスト方針をまとめて。
まだ編集はしないで。
```

PC前に戻る前に、設計の下ごしらえができます。

### 4. 進行中タスクの詰まりを解消する

```text
今のエラーの原因を調べて。
ログ、直近の差分、関連テストを見て、原因候補を優先順位付きで出して。
修正できそうなら最小変更で直して。
```

Codexが止まって質問してきたとき、スマホから追加情報を渡せます。

### 5. レビューと確認だけする

```text
今の変更について、ユーザー影響、破壊的変更、テスト不足の観点でセルフレビューして。
危ない点があれば修正案まで出して。
```

スマホ画面で全部の差分を読む代わりに、Codexへレビュー観点を明示して確認させます。

### 6. コンテンツ制作にも使う

Codexは開発だけでなく、ローカルファイルやMarkdownを扱う作業にも使えます。

```text
Downloads内の今日の動画素材を確認して、YouTube本編、Shorts、Reels、note化に使う素材候補を整理して。
ファイル削除や移動はまだしないで、まず一覧を出して。
```

動画制作、SNS展開、note記事化、投稿前チェックにも応用できます。

## スマホCodexの実践プロンプト集

そのまま使える形でまとめます。

### 初回調査

```text
このプロジェクトの構成を把握して。
主要ディレクトリ、起動方法、テスト方法、注意すべき設定ファイルをまとめて。
まだファイル編集はしないで。
```

### 小さなバグ修正

```text
このエラーを調査して、原因を特定してから最小変更で修正して。
修正後に関連テストを実行して、結果を短く報告して。
```

### UI修正

```text
スマホ表示で崩れている箇所を直して。
既存デザインに合わせて、文字の重なり、横スクロール、ボタン内テキストのはみ出しを確認して。
可能ならスクリーンショットで検証して。
```

### README整備

```text
READMEを初見の人向けに整えて。
セットアップ、起動、よくあるエラー、運用上の注意点を追加して。
コードは触らないで。
```

### PR前チェック

```text
PR前チェックをして。
差分を読み、バグリスク、テスト不足、不要な変更、秘密情報混入がないか確認して。
問題があれば修正してから、最終サマリーを出して。
```

### テスト追加

```text
今回の変更に対して不足しているテストを追加して。
既存のテストスタイルに合わせて、過剰なリファクタはしないで。
最後に実行したテストコマンドと結果を報告して。
```

### エラー調査のみ

```text
この失敗の原因を調べて。
まだ修正はしないで、原因候補、根拠、次に試すことをまとめて。
```

### 長時間タスク

```text
この機能を実装して。
まず既存実装を調査し、方針を短く共有してから編集して。
30分以上かかる場合は途中で進捗を報告して。
削除、公開、外部送信、本番反映は必ず確認して。
```

### 外出先からの作業開始

```text
今から外出するので、このタスクの下調べを進めて。
関連ファイル、実装方針、リスク、テスト方針をまとめて。
編集は必要最小限にして、判断が必要なら質問して。
```

### Codexにセルフレビューさせる

```text
今の変更をレビューして。
優先度順に、実害のあるバグ、セキュリティリスク、データ破壊リスク、テスト不足だけ指摘して。
問題がなければ、その旨と残るリスクを短くまとめて。
```

### noteやSNS展開

```text
このMarkdown記事をnote公開向けに整えて。
見出し、導入、まとめ、読者が詰まりやすい注意点を補強して。
誇張表現は避け、実務で使える資料として仕上げて。
```

### 作業ログを残す

```text
作業しながらdocs/worklog.mdに、調査したこと、変更したこと、未解決事項を残して。
最後に次回再開するときの一手も書いて。
```

## うまく使うための型

スマホからCodexへ依頼するときは、次の型が強いです。

```text
目的:
やってほしいこと:
触ってよい範囲:
触らないでほしい範囲:
確認が必要な操作:
完了条件:
最後に報告してほしいこと:
```

例:

```text
目的:
```

## Claude Coreワークのディスパッチとの比較

ここでは、Claude Coreワークの「ディスパッチ」を、複数の作業をClaude側で分解し、サブエージェントやタスクとして並列投入していく運用として比較します。

どちらが上というより、使いどころが違います。

| 観点 | Codex Mobile | Claude Coreワークのディスパッチ |
|---|---|---|
| 主な役割 | PC上のCodex作業をスマホから継続・承認・指示する | 複数タスクを分解して並列に投げる |
| 強い場面 | 外出中の確認、承認、軽い修正、長時間作業の見守り | 大きな調査、複数観点のレビュー、独立タスクの並列処理 |
| 実行場所 | Codexが動くPC、devbox、リモート環境 | Claude側のワークやサブエージェント設計に依存 |
| スマホ適性 | 高い。移動中の作業継続に向く | 指示設計はできるが、並列結果の精査は画面が小さいと重い |
| ファイル操作 | ローカル環境の実ファイルに強い | ワークの設計次第 |
| 承認フロー | コマンド、差分、外部操作をスマホから確認しやすい | ディスパッチ先の成果物統合時に確認が必要 |
| 並列性 | Codexアプリ内で複数スレッドやエージェントを扱える | ディスパッチ設計が中心 |
| 向いている人 | 外出先でもPC上の作業を止めたくない人 | 複数タスクを一気に探索・分業したい人 |

実務では、次の使い分けがわかりやすいです。

### Codex Mobileが向く作業

- 進行中の実装を外出先から確認する
- Codexの質問に答えて作業を止めない
- 小さな修正を投げる
- テスト結果や差分を確認する
- PC上の環境、ファイル、ブラウザ、ターミナルを使う
- 外出先から作業開始だけして、帰宅後にPCで仕上げる

### Claude Coreワークのディスパッチが向く作業

- 複数の案を並列で出す
- 大きなテーマを複数観点で調査する
- 設計案、競合調査、文章案などを同時に作る
- 独立した作業をサブエージェントへ切り分ける
- 1つの結論を出す前に広く探索する

### 併用すると強い流れ

おすすめは、次の流れです。

1. Claude Coreワークのディスパッチで、調査・案出し・比較を広げる
2. 採用する方針を決める
3. Codexで実装・ファイル編集・テスト・Git差分確認を進める
4. Codex Mobileで外出中も進捗確認、承認、追加指示を行う
5. 最後はPCで重要差分を確認して公開・pushする

ざっくり言うと、Claude Coreワークのディスパッチは「広げる」、Codex Mobileは「現場で動かし続ける」に強いです。

## 逆にスマホだけでやらない方がよい作業

次は、スマホだけで完結させない方が安全です。

- 本番DBの変更
- 課金設定の変更
- SNSやnoteの本番公開
- 顧客への外部送信
- 大量ファイル削除
- force push
- 認証、権限、決済まわりの大きな変更
- ライセンスや法務に関わる判断
- セキュリティ上の影響が大きい変更

スマホから依頼してもよいですが、実行前承認とPCでの最終確認を挟むのがおすすめです。

## 安全に使うためのチェックリスト

### 初期設定チェック

- ChatGPTアプリは最新版
- Codexデスクトップアプリは最新版
- スマホとPCは同じChatGPTアカウント
- Codex Mobile接続済み
- PCのスリープ対策済み
- 必要なリポジトリをCodexで開ける
- 必要な認証や環境変数がPC側にある

### 作業前チェック

- 目的を明確に書いた
- 触ってよい範囲を書いた
- 触らない範囲を書いた
- 確認が必要な操作を書いた
- 完了条件を書いた
- 最後に何を報告してほしいか書いた

### 承認前チェック

- 変更ファイルを確認した
- 削除や上書きがないか確認した
- 外部送信や公開が含まれないか確認した
- テスト結果を確認した
- Codexにリスクを説明させた
- スマホで見づらい重要差分はPCで確認した

## まとめ

Codex Mobileで詰まりやすいポイントは、スマホ側ではなくPC側にあります。

特に重要なのは次の3つです。

- PCにCodexデスクトップアプリを入れる
- Codexデスクトップアプリを最新版に更新する
- PC側の「Codex mobileを設定」から接続を進める

スマホ側で「デスクトップアプリを待っています」と出たら、そこで止まったわけではありません。

そのタイミングでPC側に移動し、左メニューの「Codex mobileを設定」を押してください。

接続後は、信頼できる自分のPCであれば、

- このコンピューターをスリープさせない
- パソコンでの利用を有効にする
- Chrome拡張機能をインストール

を有効化しておくと使いやすくなります。

そして、スマホCodexは「全部をスマホで完結する」道具ではなく、「作業を止めない」ための道具です。

外出先で調査を進める。Codexからの質問に答える。承認する。方針を変える。軽い修正を投げる。帰宅後にPCで最終確認して公開する。

この流れが一番強いです。

Codex Mobileによって、開発や業務の主戦場はPCの前だけではなくなりました。

移動中でも、外出先でも、ベッドの上でも、自分のPC上で動くCodexへ指示を出し、作業を前に進められます。

ただし、権限が強い道具だからこそ、削除、上書き、外部送信、公開、課金、本番反映は必ず確認を挟む。

ここだけ守れば、Codex Mobileはかなり実用的な仕事の相棒になります。
